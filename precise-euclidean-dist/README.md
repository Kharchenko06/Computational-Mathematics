# Задание 2: Высокоточное Евклидово расстояние (IEEE 754 Double)

**Дисциплина:** Вычислительная математика 
**Ветка:** `precise-euclidean-dist`  
**Язык:** C++20, библиотека GMP

---

## Цель

Вычислить евклидово расстояние между двумя точками в 3D:

```
dist(P1, P2) = sqrt((x2-x1)^2 + (y2-y1)^2 + (z2-z1)^2)
```

с точностью до последнего бита (не более 1 ULP от точного результата).  
Эталон — вычисление через GMP с 256-битной мантиссой.

---

## Зависимости

```bash
sudo apt install g++ make libgmp-dev   # Debian/Ubuntu
```

Требуется компилятор с поддержкой C++20 и аппаратного FMA (x86-64 с AVX2/FMA3, ARM64).

## Сборка и запуск

### Собрать все

```bash
make
```

Компилирует 6 объектных файлов библиотеки, главный исполняемый файл и 5 тестов в папку `build/`.

### Запустить все тесты

```bash
make run
```

Последовательно запускает все 5 тестов и выводит результаты в терминал.

### Запустить демо

```bash
make demo
```

Два примера из командной строки: треугольник 3-4-5 (результат = 5) и случай катастрофической отмены (результат = sqrt(3)).

```bash
make clean
```

Удаляет папку `build/`.

## Командная строка (main)

```bash
./build/euclidean_distance x1 y1 z1 x2 y2 z2
```

Числа передаются как обычные double-литералы, включая научную нотацию.

**Примеры:**

```bash
# Треугольник 3-4-5, ожидаемый результат: 5.0
./build/euclidean_distance 0 0 0 3 4 0

# Катастрофическая отмена, ожидаемый результат: sqrt(3)
./build/euclidean_distance 1000000000000000 1000000000000000 1000000000000000 \
                           1000000000000001 1000000000000001 1000000000000001

# Разные порядки величин
./build/euclidean_distance 0 0 0 1e-8 1.0 1e8
```

## Реализованные алгоритмы

### Умножение (вычисление dx^2 с точным остатком)

| Метод | Файл | Суть |
|---|---|---|
| `Naive` | `multiply.cpp` | `x*x`, остаток теряется |
| `FMA` | `multiply.cpp` | 2MultFMA: `lo = fma(x,x,-hi)` — точная ошибка за 2 операции (Handbook §5.1) |
| `Ozaki` | `multiply.cpp` | Произведение Деккера через сплит Велткампа, без FMA (Handbook §4.4) |

### Суммирование (сложение dx^2 + dy^2 + dz^2)

| Метод | Файл | Суть |
|---|---|---|
| `Naive` | `compensated_sum.cpp` | `(a+b)+c`, ошибки теряются |
| `OgitaOishi` | `compensated_sum.cpp` | VecSum по Огита-Рамп-Оиши, суммирует все 6 компонент через 2Sum (Handbook §6.3) |
| `KBN2` | `compensated_sum.cpp` | Kahan-Babuka-Neumaier с поправкой для случая `|xi| > |s|` (Handbook §6.3.2) |
| `KBN3` | `compensated_sum.cpp` | KBN + второй уровень 2Sum |
| `KBN4` | `compensated_sum.cpp` | KBN + третий уровень 2Sum |

### Извлечение корня

Обычный `std::sqrt` игнорирует члены компенсационной ошибки, что дает погрешность ~`e/(2*sqrt(s))`. Вместо него применяется один шаг Ньютона с FMA-точным вычислением остатка (Handbook §5.4.2):

```
r0       = sqrt(cs.sum)
p, delta = r0*r0 точно через fma(r0, r0, -p)
residual = (cs.sum - p) + e_total - delta
r1       = r0 + residual * 0.5 / r0
```

Результат `r1` отличается от точного значения не более чем на 1 ULP.

## Тесты

### test_multiply

Проверяет EFT-свойства алгоритмов умножения:
- `veltkamp_split`: `hi + lo == x` точно
- `naive/fma/ozaki_square`: `hi == fl(x*x)` и `|lo| < |hi|`
- FMA и Ozaki дают побитово одинаковый `lo`

```bash
./build/test_multiply
```

### test_compensated_sum

Проверяет алгоритмы суммирования относительно GMP:
- `two_sum`: `hi == a+b` точно
- все 5 алгоритмов на наборе квадратов малых чисел
- катастрофическая отмена: P1=(1e15,1e15,1e15), diff=sqrt(3)

```bash
./build/test_compensated_sum
```

### test_sqrt_refined

Сравнивает `sqrt_naive` и `sqrt_refined` с GMP-эталоном на 7 случаях. Тест считается пройденным если `sqrt_refined` дает не более 1 ULP.

```bash
./build/test_sqrt_refined
```

### test_permutation

Проверяет инвариантность результата к перестановкам (dx, dy, dz) для всех 15 комбинаций методов. Поскольку FP-сложение неассоциативно, `naive_sum` может давать разные результаты при разном порядке слагаемых.

```bash
./build/test_permutation
```

### test_euclidean_full

Сводная таблица: 12 тестовых пар точек, для каждой — все 15 комбинаций (3 умножения x 5 суммирований) vs GMP-эталон. Тест проходит если хотя бы одна комбинация дает не более 1 ULP.

```bash
./build/test_euclidean_full
```

Ожидаемый итог:

```
Итог: 12/12 прошли (<= 1 ULP)
```

## Флаги компиляции

| Флаг | Причина |
|---|---|
| `-fno-unsafe-math-optimizations` | Запрещает перестановку FP-операций, что сломало бы сплит Велткампа |
| `-ffp-contract=on` | FMA применяется только там, где написан `std::fma`, не автоматически |
| `-march=native` | Включает аппаратный FMA (нужен для `fma_square`) |
| `-O2` | Стандартная оптимизация без нарушения FP-семантики |
